# Lit Review Citation Matrix 양식

> 이 템플릿은 `Lit_Review_Citation_Matrix_v2.xlsx`의 16컬럼 구조를 정확히 반영합니다.
> **핵심 원칙: 1인용 = 1행**. 동일 논문에서 여러 인용구를 추출하면 인용구마다 별도 행을 생성합니다.

## 엑셀 컬럼 구조 (16컬럼)

### 카테고리 1: 필러 연결 (A~B)

| 컬럼 | 필드명 | 설명 | 예시 |
|------|--------|------|------|
| A | Pillar | 필러-키워드 매핑 시트 기반 자동 선택 | `Pillar B: AI Teammate` |
| B | 검색 키워드 | 해당 필러의 하위 검색 키워드 | `AI as Teammate + CSCL` |

### 카테고리 2: 기본 정보 (C~F)

| 컬럼 | 필드명 | 설명 | 예시 |
|------|--------|------|------|
| C | ID / 분류 | `{필러코드}-{키워드코드}-{일련번호}{알파벳}` | `PB-AT-001a` |
| D | 저자 (연도) | APA 형식 | `Ouyang & Zhang (2024)` |
| E | 논문 제목 | 원문 제목 (동일 논문 2행째부터 `(동일 논문 — 인용구 N)`) | |
| F | 학술지/출처 | Journal명 또는 Conference명 | `Educational Research Review, 44` |

### 카테고리 3: 핵심 내용 (G~I)

| 컬럼 | 필드명 | 설명 | 예시 |
|------|--------|------|------|
| G | 연구 목적 | 이 논문이 해결하려는 문제 (1문장) | |
| H | 연구 방법론 | 연구유형, 참여자수, 분석방법 | `체계적 문헌리뷰(PRISMA), N=26` |
| I | 주요 결과 (요약) | 핵심 발견 요약 (인용구별로 관련 결과 기술) | |

### 카테고리 4: 정밀 인용 (J~L)

| 컬럼 | 필드명 | 설명 | 예시 |
|------|--------|------|------|
| J | 핵심 인용구 (Direct Quote) | **원문 그대로** 큰따옴표로 감싸 복사 | `"These student-facing..."` |
| K | 출처 위치 (p.#, L.#) | 페이지, 줄/문단 번호 | `p.6, para.2` |
| L | 인용 유형 | `근거(Support)` / `대조(Contrast)` / `Gap` / `방법론참조(Method)` | `Gap` |

### 카테고리 5: 비평 및 연결 (M~N)

| 컬럼 | 필드명 | 설명 | 예시 |
|------|--------|------|------|
| M | 핵심 키워드 | 필터링용 태그 (쉼표 구분) | `CSCL, AI, feedback` |
| N | 한계점/후속연구 | 저자가 밝힌 한계 + 내가 발견한 빈틈 | |

### 카테고리 6: 관리 상태 (O~P)

| 컬럼 | 필드명 | 설명 | 예시 |
|------|--------|------|------|
| O | 진행 상태 | `읽기 전` / `읽는 중` / `인용구 추출 중` / `완료` | `완료` |
| P | 파일 링크 | 원본 PDF 경로 | `/Lit Review/Pillar_B/...` |

## ID 생성 규칙

`{필러코드}-{키워드코드}-{일련번호}{알파벳}`

- **필러코드**: PA, PB, PC, PD (Pillar A~D)
- **키워드코드**: 필러-키워드 매핑 시트의 ID 접두어 참조 (예: AT, SM, CS...)
- **일련번호**: 해당 키워드 내 논문 순번 (001, 002...)
- **알파벳**: 동일 논문 내 인용구 순번 (a, b, c...)

예시: `PB-AT-001a` → Pillar B, AI Teammate 키워드, 1번째 논문, 첫 번째 인용구

## 동일 논문 복수 인용 행 규칙

1. **첫 번째 행 (001a)**: 모든 16컬럼 완전 기입
2. **두 번째 행 이후 (001b, 001c...)**:
   - A~B: 동일 (필러/키워드)
   - C: ID 알파벳만 증가
   - D: 동일 저자
   - E: `(동일 논문 — 인용구 N)` 표기
   - F: 비워둠
   - G~H: 비워둠
   - I: **해당 인용구와 관련된 결과만** 기술
   - J~L: 새 인용구 정보
   - M~N: 비워둠 (첫 행에만 기입)
   - O: 동일
   - P: 비워둠

## 인용 유형 기준

| 유형 | 의미 | Literature Review에서의 활용 |
|------|------|---------------------------|
| **근거(Support)** | 우리 논문의 주장/가설을 뒷받침하는 근거 | "선행연구에 따르면..." 형태로 활용 |
| **대조(Contrast)** | 우리 주장과 상반되는 결과/관점 | "반면에..." "그러나..." 형태로 활용 |
| **Gap** | 선행연구의 공백, 미해결 문제 | "아직 ~는 밝혀지지 않았다" 형태로 활용 |
| **방법론참조(Method)** | 방법론적으로 참고할 설계/절차 | "본 연구는 ~의 방법론을 차용하여" 형태로 활용 |

## 마크다운 출력 양식

```markdown
| Pillar | 검색 키워드 | ID | 저자(연도) | 논문 제목 | 학술지 | 연구 목적 | 방법론 | 주요 결과 | 핵심 인용구 | 출처 위치 | 인용 유형 | 키워드 | 한계점 | 상태 | 파일 |
|--------|-----------|-----|----------|----------|--------|----------|--------|----------|-----------|----------|----------|--------|--------|------|------|
| ... | ... | ... | ... | ... | ... | ... | ... | ... | ... | ... | ... | ... | ... | ... | ... |
```
