# 정밀 인용 추출 형식 (Citation Extraction Format)

> `Lit_Review_Citation_Matrix_v2.xlsx`의 정밀 인용 컬럼(J~L)에 직접 매핑되는 형식입니다.
> **목적**: 학술 논문 Literature Review 작성을 위한 인용구 추출 및 분류.

## Strict Verbatim 원칙

인용구는 원문 그대로 추출합니다. **한 글자의 오차도 불허합니다.**
- 큰따옴표(`"..."`)로 감싸 원문 복사
- 줄임(`...`)은 논문 원문에 있는 경우에만 사용
- 맞춤법 오류도 원문 그대로 보존 (필요시 `[sic]` 표기)

## 인용 추출 단위

**1인용 = 1행**. 하나의 논문에서 여러 인용구를 추출하면, 인용구마다 별도 행(엑셀 row)을 생성합니다.

## 인용 출력 형식

각 인용구에 대해 다음 3개 필드를 추출합니다:

### J: 핵심 인용구 (Direct Quote)
```
"원문 문장을 큰따옴표로 감싸 그대로 복사"
```

### K: 출처 위치 (p.#, L.#)
```
p.{페이지번호}, {위치유형}.{번호}
```
위치유형: `L.` (줄번호), `para.` (문단번호), `section` (섹션명)

예시:
- `p.6, para.2`
- `p.8, L.14-16`
- `p.11, section 5.4, para.2`

### L: 인용 유형
다음 4가지 중 하나를 반드시 선택합니다:

| 유형 | 코드 | 판단 기준 | Literature Review 활용 |
|------|------|----------|----------------------|
| **근거** | `근거(Support)` | 우리 연구의 주장·가설·전제를 뒷받침 | "~에 따르면", "~가 확인한 바와 같이" |
| **대조** | `대조(Contrast)` | 우리 주장과 상반되거나 다른 관점 제시 | "반면에", "그러나 ~는 다른 결과를 보고" |
| **Gap** | `Gap` | 선행연구의 공백, 미해결 문제, 한계 지적 | "아직 ~는 밝혀지지 않았다", "~가 부족하다" |
| **방법론참조** | `방법론참조(Method)` | 연구 설계·분석 방법을 참고할 가치 | "본 연구는 ~의 방법론을 차용하여" |

## 인용 유형 판단 가이드

프로젝트 맥락(Pillar, 검색 키워드, 연구 목적)을 기준으로 판단합니다:

1. **이 인용구가 우리 연구의 필요성(rationale)을 뒷받침하는가?** → `근거(Support)`
2. **이 인용구가 우리 연구와 다른 결과/해석을 제시하는가?** → `대조(Contrast)`
3. **이 인용구가 "아직 연구되지 않은 영역"을 지적하는가?** → `Gap`
4. **이 인용구의 연구 방법이 우리 연구에 참고할 만한가?** → `방법론참조(Method)`

## 인용구 선별 기준

Literature Review 작성에 실질적으로 활용 가능한 인용구만 추출합니다:

1. **연구 목적/가설과 직접 관련**되는 문장
2. **통계적 수치나 구체적 증거**가 포함된 문장
3. **선행연구의 한계나 Gap**을 명시적으로 언급하는 문장
4. **이론적 프레임워크의 핵심 정의**를 담은 문장
5. **방법론적 설계의 근거**가 되는 문장

불필요한 인용구 (배경 설명, 일반 상식, 결론의 반복)는 제외합니다.

## 복수 인용 시 ID 규칙

동일 논문에서 N개의 인용구를 추출하면:
- 첫 번째 인용구: `{접두어}-{일련번호}a`
- 두 번째 인용구: `{접두어}-{일련번호}b`
- N번째 인용구: `{접두어}-{일련번호}{n번째 알파벳}`

예: PB-AT-001a, PB-AT-001b, PB-AT-001c, ...
