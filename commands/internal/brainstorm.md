# /gadfly-brainstorm

러프한 연구 아이디어를 Gadfly AI Lab의 브레인스토머와 기획전문가가 병렬로 분석하여 학술적으로 구체화합니다.

## 사용법

```
/gadfly-brainstorm <러프한 아이디어>
/gadfly-brainstorm <러프한 아이디어> --project <기존 프로젝트와 연결 시>
```

## 실행 절차

### Phase 1: 소장 - 아이디어 접수

1. 러프한 아이디어를 분석합니다.
2. 관련 프로젝트 설정이 있으면 `projects/<프로젝트명>.md`를 읽어 맥락을 파악합니다.
3. `knowledge/core/korean-academic-conventions.md`를 참조하여 한국 학술 맥락을 확인합니다.
4. **브레인스토머(ideation)**와 **기획전문가(planning)**를 병렬 소집합니다.
5. 필요 시 **검색전략가(search-strategist)**도 추가 배치합니다.

### Phase 2: 전문가 병렬 분석

두 전문가가 동시에 아이디어를 분석합니다:

- **브레인스토머 (gadfly-ideation)**:
  - 아이디어를 학술적 언어로 정제
  - 이론 프레임워크 선택 + 정당화 (왜 이 이론인가?)
  - 개념의 조작적 정의 (측정 가능한 정의)
  - 연구 질문(RQ) 도출
  - 연구 갭 식별

- **기획전문가 (gadfly-planning)**:
  - 온톨로지 맵으로 개념 관계도 구축
  - 탐색해야 할 상위/인접 개념 도출
  - 연구 갭을 관계 부재로 식별
  - 연구 포지셔닝 전략 제안
  - 예상 리뷰어 비판 시뮬레이션

전문가들은 **병렬로 호출**합니다.

### Phase 3: 소장 - 통합 보고서 작성

전문가 의견을 종합하여 최종 산출물을 생성합니다:

```markdown
# 연구 아이디어 브레인스토밍 보고서

## 1. 아이디어 정제
- **원본**: (설립자 원문 그대로)
- **학술적 재정의**: (학술적 언어로 재구성)
- **예상 연구 질문**:
  1. RQ1: ...
  2. RQ2: ...
  3. RQ3: ...

## 2. 이론 프레임워크 정당화
| 이론/모델 | 선택 근거 | 핵심 개념 | 조작적 정의 |
|-----------|-----------|-----------|------------|

## 3. 개념 온톨로지 맵
(기획전문가의 온톨로지 트리플 기반 개념 관계 구조)

## 4. 전문가별 관점 요약
| 전문가 | 핵심 의견 | 추천 탐색 방향 |
|--------|----------|---------------|
| 브레인스토머 | | |
| 기획전문가 | | |

## 5. 연구 포지셔닝 제안
- 기존 연구 지형 예측
- 차별화 포인트
- 예상 기여(Contribution): 이론적/방법론적/실천적

## 6. 예상 리뷰어 비판 시뮬레이션
| 예상 비판 | 대응 전략 |
|-----------|-----------|

## 7. 다음 단계 로드맵
- Step 1: /gadfly-search-strategy로 검색 전략 수립
- Step 2: 논문 수집
- Step 3: /gadfly-cite로 정밀 인용 분석
- Step 4: /gadfly-methodology로 방법론 설계
```

보고서를 `output/<프로젝트 또는 brainstorm>/brainstorm-YYYY-MM-DD.md`로 저장합니다.
