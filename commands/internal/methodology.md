# /gadfly-methodology

대화형 인터뷰를 통해 연구 질문에 가장 적합한 방법론을 추천하고, 설문 설계, IRB 절차, 재현가능성 확보를 안내합니다.

## 사용법

```
/gadfly-methodology --project <프로젝트명>
/gadfly-methodology <연구질문 또는 주제>
```

## 실행 절차

### Phase 1: 소장 - 대화형 인터뷰

프로젝트 설정이 있으면 먼저 읽고, 없거나 불충분한 정보는 대화형으로 수집합니다:

1. **연구 질문**: "어떤 연구 질문에 답하고 싶으신가요?"
2. **연구 목적**: "탐색적/기술적/설명적/인과적 연구 중 어디에 해당하나요?"
3. **대상/맥락**: "연구 대상은 누구이고, 어떤 맥락인가요?"
4. **데이터 접근성**: "어떤 데이터를 수집할 수 있나요? (설문, 인터뷰, 관찰, 기존 데이터 등)"
5. **제약 조건**: "시간, 예산, 참여자 접근성 등 제약이 있나요?"

AskUserQuestion 도구를 활용하여 효율적으로 정보를 수집합니다.

### Phase 2: 방법론 전문가 - 추천 및 설계

수집된 정보를 **방법론 전문가(methodology)**에게 전달합니다:

- 프로젝트 설정 + 인터뷰 결과 전달
- `knowledge/core/research-methodology.md` 참조 지시
- `knowledge/core/survey-design.md` 참조 지시 (설문 포함 시)
- `knowledge/core/statistical-methods.md` 참조 지시
- 이론↔방법론 정합성 검증 요청
- IRB 체크리스트 포함 요청
- 재현가능성 확보 수준 상세 기술 요청

### Phase 3: 소장 - 보고서 출력

```markdown
# 방법론 추천 보고서

## 프로젝트: [프로젝트명]

## 1. 연구 설계 추천
| 항목 | 내용 |
|------|------|
| 추천 설계 | (양적/질적/혼합/DBR) |
| 구체적 방법 | (실험/준실험/사례연구/문화기술지 등) |
| 선택 근거 | |

## 2. 이론-방법론 정합성
| 이론 프레임워크 | 방법론 | 정합성 근거 |
|----------------|--------|------------|

## 3. 데이터 수집 계획
- **참여자**: 대상, 표본 크기, 샘플링 전략
- **도구**: 설문지/인터뷰 프로토콜/관찰 체크리스트
- **절차**: 단계별 수집 절차

## 4. 설문/도구 설계 (해당 시)
- 척도 구성 및 문항 예시
- 타당도/신뢰도 확보 전략
- 파일럿 테스트 계획

## 5. 분석 계획
- 분석 방법 및 도구 (R/SPSS/Python)
- 가정 검증 절차
- 보고 형식 (APA 7th)

## 6. IRB 체크리스트
- [ ] 연구 목적 명시
- [ ] 자발적 참여 보장
- [ ] 비밀보장 조치
- [ ] 철회 권리 고지
- [ ] 동의서 준비
- [ ] 취약 집단 보호 (해당 시)
- [ ] 데이터 보관/폐기 계획

## 7. 재현가능성 확보
(다른 연구자가 동일 절차를 재현할 수 있는 수준의 상세 기술)

## 8. 한계 및 대안
| 잠재 한계 | 대안/완화 전략 |
|-----------|---------------|
```

보고서를 `output/<프로젝트>/methodology-YYYY-MM-DD.md`로 저장합니다.
