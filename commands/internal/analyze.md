# /gadfly-analyze

논문 한 편을 Gadfly AI Lab의 기능별 전문가들이 다각도로 종합 분석합니다.

## 사용법

```
/gadfly-analyze <논문 파일 경로> --project <프로젝트명>
```

## 실행 절차

### Phase 1: 소장 - 접수 및 메타데이터 추출

1. 논문 파일을 Read 도구로 읽습니다.
2. 다음 메타데이터를 추출합니다:
   - 제목, 저자(연도), 학술지, DOI
   - 섹션 구조 (서론, 문헌고찰, 방법론, 결과, 논의 등)
3. `projects/<프로젝트명>.md` 파일을 읽어 프로젝트 맥락을 파악합니다.
4. `templates/lit-review-template.md`를 읽어 출력 양식을 확인합니다.
5. 논문 내용과 프로젝트 키워드를 대조하여 필요한 전문가를 판단합니다.

### Phase 2: 전문가 배치 및 분석

판단된 전문가들에게 Task 도구를 사용하여 분석을 지시합니다. 각 전문가에게 다음을 전달합니다:
- 논문 내용
- 프로젝트 설정 (키워드, 이론적 프레임워크, 비판 프레임)
- 해당 전문가의 knowledge 폴더 참조 지시

**전문가 배치 매트릭스**:
| 요청 유형 | 필수 배치 | 권장 배치 |
|-----------|-----------|-----------|
| 논문 종합 분석 | literature-analyst, planning | methodology, data-analyst |

사용 가능한 전문가:
- `gadfly-ideation`: 브레인스토머 — 이론 프레임워크 연결, 개념 정의
- `gadfly-search-strategist`: 검색전략가 — 추가 검색 방향 제안
- `gadfly-literature-analyst`: 문헌분석가 — **정밀 인용 추출** (필수)
- `gadfly-methodology`: 방법론 전문가 — 방법론 평가
- `gadfly-data-analyst`: 데이터 분석가 — 결과/통계 평가
- `gadfly-planning`: 기획전문가 — 온톨로지 구조화, 전략적 연결

독립적인 분석은 **병렬로 호출**합니다.

### Phase 3: 결과 취합 및 Lit Review 양식 출력

1. 전문가들의 분석 결과를 수집합니다.
2. 중복되는 인용 후보를 통합합니다.
3. `templates/lit-review-template.md` 양식으로 정제합니다:

```markdown
## 분석 결과: [논문 제목]

### 1. 기본 정보
| 항목 | 내용 |
|------|------|
| ID/분류 | |
| 저자(연도) | |
| 논문 제목 | |
| 학술지/출처 | |
| DOI | |

### 2. 핵심 내용
| 항목 | 내용 |
|------|------|
| 연구 목적 | |
| 연구 방법론 | |
| 주요 결과 | |
| 이론적 프레임워크 | |

### 3. 비평 및 연결
| 항목 | 내용 |
|------|------|
| 핵심 키워드 | |
| 한계점/후속연구 | |
| 내 논문과의 연관성 | |
| 방법론적 품질 평가 | |

### 4. 전문가별 분석
(각 전문가의 출력을 정리)

### 5. 정밀 인용 후보
| # | Location | Original Text | Reason | 통합 제안 |
|---|----------|---------------|--------|----------|

### 6. 온톨로지 트리플 (기획전문가)
(해당 시)

### 7. 한계점 종합
(전문가별 한계점을 종합 정리)
```

4. 결과를 `output/<프로젝트>/` 폴더에 마크다운 파일로 저장합니다.
5. 터미널에 요약 결과를 출력합니다.
