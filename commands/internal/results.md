# /gadfly-results

연구 데이터 또는 결과를 분석하여 APA 형식의 결과(Results) 섹션, 고찰(Discussion) 초안, 기여(Contribution) 정리까지 포괄합니다.

## 사용법

```
/gadfly-results --project <프로젝트명>
/gadfly-results <데이터 파일 경로> --project <프로젝트명>
```

## 실행 절차

### Phase 1: 소장 - 데이터 접수

1. 데이터 파일이 제공되면 Read 도구로 읽습니다 (CSV, 통계 출력, 텍스트 등).
2. 데이터가 없으면 사용자에게 분석 결과를 직접 입력하도록 안내합니다.
3. `projects/<프로젝트명>.md`를 읽어 연구 맥락을 파악합니다:
   - 연구 질문(RQ)
   - 이론적 프레임워크
   - 선행연구 맥락
4. 기존 `output/<프로젝트>/` 폴더에서 이전 분석 결과 (methodology 보고서 등)를 확인합니다.

### Phase 2: 데이터 분석가 - 분석 및 보고

**데이터 분석가(data-analyst)**에게 Task를 지시합니다:

- 데이터/결과 + 프로젝트 맥락 전달
- `knowledge/core/statistical-methods.md` 참조 지시
- `knowledge/core/academic-writing.md` 참조 지시
- `knowledge/core/korean-academic-conventions.md` 참조 지시
- APA 7th 형식 결과 작성 요청
- 고찰(Discussion) 초안 작성 요청
- 기여(Contribution) 정리 요청

필요 시 **방법론 전문가(methodology)**도 추가 배치하여 결과 해석의 방법론적 타당성을 검증합니다.

### Phase 3: 소장 - 결과 보고서 출력

```markdown
# 결과 분석 보고서

## 프로젝트: [프로젝트명]

## 1. Results (결과)

### 1.1 기술 통계
(평균, 표준편차, 빈도 등 — APA 형식 테이블)

| 변인 | M | SD | N |
|------|---|----|----|

### 1.2 추론 통계
(가설 검정 결과 — APA 형식)

| 가설/RQ | 분석 방법 | 결과 | 효과크기 | p |
|---------|----------|------|---------|---|

### 1.3 효과크기 해석
(Cohen's d, η², f² 등의 해석)

### 1.4 표/그림
(APA 형식 표/그림 제안)

---

## 2. Discussion (고찰)

### 2.1 주요 발견 요약
(핵심 결과를 연구 질문별로 요약)

### 2.2 선행연구와의 비교
(결과가 기존 연구를 지지/반박/확장하는지)

### 2.3 이론적 함의
(결과가 이론 프레임워크에 주는 시사점)

### 2.4 실천적 함의
(교육/산업/정책 현장에 대한 시사점)

### 2.5 한계
(연구의 제한점 — 일반화 한계, 방법론적 한계 등)

### 2.6 후속 연구 제안
(한계를 보완할 수 있는 후속 연구 방향)

---

## 3. Contribution (기여)

### 이론적 기여
(학문적 지식에 대한 새로운 기여)

### 방법론적 기여
(연구 방법에 대한 기여 — 해당 시)

### 실천적 기여
(현장 실천에 대한 기여)

---

## 4. 출판 전략 (선택)

| 항목 | 내용 |
|------|------|
| 추천 학술지 | (KCI/SSCI 등) |
| 투고 전략 | |
| 예상 리비전 포인트 | |
```

보고서를 `output/<프로젝트>/results-YYYY-MM-DD.md`로 저장합니다.
