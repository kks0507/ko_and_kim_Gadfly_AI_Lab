# /gadfly-cite

논문 한 편에서 정밀 인용구를 추출하고, Lit Review 양식에 맞춰 구조화된 분석 결과를 출력합니다.

## 사용법

```
/gadfly-cite <논문 파일 경로> --project <프로젝트명>
```

## 실행 절차

### Phase 1: 소장 - 접수 및 메타데이터 추출

1. 논문 파일을 Read 도구로 읽습니다.
2. 다음 메타데이터를 추출합니다:
   - 제목, 저자(연도), 학술지, DOI
   - 섹션 구조 (서론, 문헌고찰, 방법론, 결과, 논의 등)
3. `projects/<프로젝트명>.md` 파일을 읽어 프로젝트 맥락을 파악합니다.
4. `templates/lit-review-template.md`를 읽어 출력 양식을 확인합니다.
5. `templates/citation-template.md`를 읽어 인용 형식을 확인합니다.

### Phase 2: 문헌분석가 - 정밀 인용 추출

**문헌분석가(literature-analyst)**에게 Task를 지시합니다:

- 논문 내용 전체를 전달
- 프로젝트 설정 (키워드, 이론적 프레임워크, 비판 프레임) 전달
- `templates/citation-template.md` 형식에 따른 정밀 인용 추출 지시
- 인용 통합 기법 제안 (패러프레이즈 vs 직접 인용) 포함
- 문헌의 방법론적 품질 평가 포함

### Phase 3: 소장 - Lit Review 양식 출력

`templates/lit-review-template.md` 양식에 맞춰 최종 결과를 정리합니다:

```markdown
## Lit Review 양식: [논문 제목]

### 1. 기본 정보
| 항목 | 내용 |
|------|------|
| ID/분류 | (프로젝트-순번) |
| 저자(연도) | |
| 논문 제목 | |
| 학술지/출처 | |
| DOI | |

### 2. 핵심 내용
| 항목 | 내용 |
|------|------|
| 연구 목적 | |
| 연구 방법론 | |
| 주요 결과 | |
| 이론적 프레임워크 | |

### 3. 비평 및 연결
| 항목 | 내용 |
|------|------|
| 핵심 키워드 | |
| 한계점/후속연구 | |
| 내 논문과의 연관성 | |
| 방법론적 품질 평가 | (상/중/하 + 근거) |

### 4. 관리 상태
| 항목 | 내용 |
|------|------|
| 진행 상태 | 분석완료 |
| 파일 링크 | (원본 경로) |

### 5. 정밀 인용

#### 인용 #1
- **Location**: P.X, Line X
- **Original Text**: "원문 그대로"
- **앞뒤 맥락**: ...
- **인용 사유**: ...
- **통합 제안**: 패러프레이즈 추천 / 직접 인용 추천

(필요한 만큼 반복)
```

4. 결과를 `output/<프로젝트>/cite-[논문약칭]-YYYY-MM-DD.md`로 저장합니다.
5. 터미널에 기본 정보 + 인용 수 요약을 출력합니다.
