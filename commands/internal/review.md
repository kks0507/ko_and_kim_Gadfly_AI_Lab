# /gadfly-review

프로젝트에 축적된 분석 결과들을 종합하여 문헌 리뷰 보고서를 생성합니다.

## 사용법

```
/gadfly-review <주제 또는 키워드> --project <프로젝트명>
```

## 실행 절차

### Phase 1: 소장 - 기존 분석 결과 수집

1. `output/<프로젝트>/` 폴더의 모든 분석 결과를 스캔합니다.
2. 주제/키워드와 관련된 분석 결과를 필터링합니다.
3. 프로젝트 설정 파일을 참조하여 맥락을 파악합니다.
4. `knowledge/core/academic-writing.md`를 참조하여 리뷰 구조를 확인합니다.

### Phase 2: 기획전문가 + 문헌분석가 - 구조화

**기획전문가(planning)**에게 Task를 지시하여:
- 기존 분석들의 개념 온톨로지를 통합
- 논문 간 관계 (지지/반박/확장/보완)를 식별
- 연구 흐름의 시간적 변화를 파악
- 크로스-프로젝트 연결 (다른 프로젝트의 관련 분석도 참조)

**문헌분석가(literature-analyst)**에게 Task를 지시하여:
- 킬러 인용구 선별 및 통합 제안 (패러프레이즈/직접 인용)
- 문헌 간 방법론적 품질 비교

### Phase 3: 소장 - 종합 리뷰 보고서

```markdown
# 문헌 리뷰 종합 보고서

## 프로젝트: [프로젝트명]
## 주제: [주제/키워드]
## 분석 대상: 총 N편

## 1. 연구 동향 개요
(시간 순서별 연구 흐름 정리)

## 2. 주요 이론적 프레임워크
(분석된 논문들이 사용한 이론들 분류)

## 3. 방법론 비교
| 저자(연도) | 연구 설계 | 참여자 | 분석 방법 | 품질 평가 |
|-----------|----------|--------|----------|----------|

## 4. 핵심 발견 종합
(전문가별 관점에서 도출된 주요 발견들을 통합)

## 5. 통합 온톨로지 맵
(기획전문가의 종합 개념 관계도)
- 핵심 개념 간 관계 트리플
- 논문 간 개념 교차점
- 미탐색 관계 (연구 갭)

## 6. 연구 갭 분석
- 아직 탐색되지 않은 영역
- 상충되는 결과들
- 방법론적 한계 패턴

## 7. 킬러 인용구 목록
(전체 분석에서 가장 중요한 인용 후보들)
| # | 저자 | Location | Original Text | Reason | 통합 제안 |
|---|------|----------|---------------|--------|----------|

## 8. 후속 연구 제안
- 추가 수집이 필요한 논문 방향
- 우리 연구의 차별화 전략 강화 방안
- 추천 검색 키워드 (추가 수집용)
```

보고서를 `output/<프로젝트>/review-YYYY-MM-DD.md`로 저장합니다.
