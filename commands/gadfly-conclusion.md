# /gadfly-conclusion — 결론 및 고찰

연구의 마무리 단계를 통합 지원합니다. 기존 분석 결과를 바탕으로 고찰, 한계점, 후속 연구 제안, 기여 진술, 출판 전략을 종합 생성합니다.

## 사용법

```
/gadfly-conclusion --project <프로젝트명>
```

## 하위 기능 트리

Director가 프로젝트 맥락과 기존 결과를 분석하여 전문가를 배치합니다:

| 하위 기능 | 트리거 조건 | 배치 에이전트 |
|-----------|------------|-------------|
| 고찰 초안 | 항상 | data-analyst + planning |
| 한계점 분석 | 항상 | data-analyst + methodology |
| 후속 연구 제안 | 항상 | planning + ideation |
| 기여 진술 | 항상 | data-analyst |
| 출판 전략 | 요청 시 또는 투고 단계일 때 | planning + data-analyst |

## 실행 절차

### Phase 1: Director — 맥락 수집

1. `output/<프로젝트>/results-*.md`를 읽어 결과 분석 결과를 파악합니다.
2. `projects/<프로젝트명>.md`를 읽어 프로젝트 설정을 확인합니다:
   - 이론적 프레임워크
   - 연구 질문(RQ)
   - 독창적 주장
   - 비판 프레임
   - 현재 연구 단계
3. `output/<프로젝트>/` 폴더에서 관련 분석 결과를 추가 수집합니다:
   - `methodology-*.md` — 방법론 보고서
   - `review-*.md` — 문헌 리뷰 보고서
   - `plan-*.md` — 연구 계획 보고서
4. `knowledge/core/academic-writing.md`를 참조합니다.
5. `knowledge/core/korean-academic-conventions.md`를 참조합니다.

### Phase 2: 전문가 병렬 배치

다음 전문가들을 **병렬**로 배치합니다:

#### A. 데이터 분석가 (gadfly-data-analyst)
- 결과 해석: 주요 발견의 의미와 맥락
- 선행연구 비교: 기존 연구 결과와의 일치/불일치 분석
- 이론적 함의: 결과가 이론 프레임워크에 주는 시사점
- 실천적 함의: 교육/산업/정책 현장에 대한 시사점
- 기여 진술: 이론적/방법론적/실천적 기여 정리

#### B. 기획전문가 (gadfly-planning)
- 예상 리뷰어 비판 시뮬레이션 (3-5개 비판 + 대응)
- 출판 전략: 추천 학술지 (KCI/SSCI/Scopus), 투고 전략
- 리비전 대응 전략: 예상 리비전 포인트 + 대응 초안
- 연구 포트폴리오 관점: 후속 연구와의 연결고리

#### C. 방법론전문가 (gadfly-methodology)
- 방법론적 한계 분석: 연구 설계, 표본, 도구의 제한점
- 일반화 한계: 외적 타당도 이슈
- 대안 방법론 제안: 한계를 보완할 수 있는 후속 방법론

#### D. 브레인스토머 (gadfly-ideation)
- 후속 연구 방향 제안: 연구 갭 기반
- 확장 가능한 연구 주제 아이디어
- 학제간 연결 가능성

### Phase 3: Director — 통합 보고서 출력

전문가 결과를 종합하여 최종 보고서를 생성합니다:

```markdown

---

|----|----------|------|

| 선행연구 | 본 연구 결과 | 관계 (지지/반박/확장) | 해석 |
|----------|------------|---------------------|------|

### 1.3 이론적 함의
(결과가 이론 프레임워크에 기여하는 바)

### 1.4 실천적 함의
(교육/산업/정책 현장에 대한 시사점)

---

## 2. Limitations (한계점)

### 2.1 방법론적 한계
| 한계 | 영향 | 완화 조치 |
|------|------|----------|

### 2.2 일반화 한계
(외적 타당도 이슈)

### 2.3 기타 한계
(시간, 자원, 맥락적 제한)

---

## 3. Future Research (후속 연구 제안)

(본 연구의 한계를 보완하는 연구)

| # | 후속 연구 주제 | 보완하는 한계 | 제안 방법론 |
|---|---------------|-------------|-----------|

### 3.2 확장 연구
(본 연구를 다른 맥락/대상으로 확장)

### 3.3 학제간 연결
(다른 분야와의 협력 가능성)

---

## 4. Contribution (기여 진술)

(학문적 지식에 대한 새로운 기여)

### 4.2 방법론적 기여
(연구 방법에 대한 기여 — 해당 시)

### 4.3 실천적 기여
(현장 실천에 대한 기여)

---

## 5. Publication Strategy (출판 전략)

### 5.1 추천 학술지
| 순위 | 학술지명 | 등급 | 적합도 | 비고 |
|------|---------|------|--------|------|
| 1 | | KCI/SSCI/Scopus | | |
| 2 | | | | |
| 3 | | | | |

- 투고 전 점검 사항
- 커버 레터 핵심 포인트
- 에디터에게 어필할 포인트

### 5.3 예상 리뷰어 비판 및 대응
| # | 예상 비판 | 심각도 | 대응 전략 |
|---|----------|--------|----------|
| 1 | | Major/Minor | |
| 2 | | | |

### 5.4 리비전 대응 전략
(리비전 요청 시 대응 프레임워크)
```

보고서를 `output/<프로젝트>/conclusion-YYYY-MM-DD.md`로 저장합니다.
