# /gadfly-ideate

러프한 연구 아이디어를 Gadfly AI Lab 전문가 전원이 구체화하고, 선행연구 조사 전략까지 수립합니다.

## 사용법

```
/gadfly-ideate <러프한 아이디어>
/gadfly-ideate <러프한 아이디어> --project <기존 프로젝트와 연결 시>
```

## 실행 절차

### Phase 1: 소장 - 아이디어 접수 및 전문가 전원 소집

1. 러프한 아이디어를 분석합니다.
2. 관련 프로젝트 설정이 있으면 참조합니다.
3. **전문가 전원을 소집**합니다 (ideate는 다각적 관점이 핵심).

### Phase 2: 전문가별 아이디어 해석

각 전문가가 자기 렌즈로 아이디어를 해석합니다:

- **AI시스템 전문가**: "기술적으로 이런 접근이 가능합니다"
- **교육공학 전문가**: "이론적 근거는 이쪽에서 찾아야 합니다"
- **문헌정보 전문가**: "정보학 관점에선 이런 선행연구가 있을 것입니다"
- **교육전문가**: "현장 적용 시 이런 맥락을 고려해야 합니다"
- **기획전문가**: 온톨로지 맵으로 개념 관계도를 구축합니다

전문가들은 **병렬로 호출**합니다.

### Phase 3: 소장 - 종합 보고서 작성

전문가 의견을 종합하여 최종 산출물을 생성합니다:

```markdown
# 연구 아이디어 구체화 보고서

## 1. 아이디어 정제
- **원본**: (설립자 원문 그대로)
- **구체화**: (학술적 언어로 재정의)
- **예상 연구 질문**:
  1. RQ1: ...
  2. RQ2: ...
  3. RQ3: ...

## 2. 개념 온톨로지 맵
(기획전문가의 온톨로지 트리플 기반)
- 핵심 개념 간 관계 구조
- 탐색해야 할 상위/인접 개념

## 3. 전문가별 관점 요약
| 전문가 | 핵심 의견 | 추천 탐색 방향 |
|--------|----------|---------------|

## 4. 선행연구 조사 전략
- **추천 검색 키워드** (영/한)
- **추천 데이터베이스**: ERIC, LISTA, Scopus, Web of Science, RISS 등
- **검색 우선순위**: (어떤 키워드부터, 왜)
- **예상 필요 논문 수 및 분류 기준**

## 5. 연구 포지셔닝 제안
- 기존 연구 지형 예측
- 우리가 노릴 수 있는 갭/차별화 포인트
- 추천 이론적 프레임워크

## 6. 다음 단계 로드맵
- Step 1: 이 키워드로 논문 수집
- Step 2: /gadfly-batch로 일괄 분석
- Step 3: /gadfly-review로 종합
```

보고서를 `output/<프로젝트 또는 ideate>/ideate-YYYY-MM-DD.md`로 저장합니다.
