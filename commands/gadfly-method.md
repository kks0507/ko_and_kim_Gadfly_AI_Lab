# /gadfly-method — 연구 방법

연구 방법 설계의 모든 과정을 통합 지원합니다. Director가 대화형 인터뷰로 연구 맥락을 파악한 후, 필요한 하위 기능을 자율적으로 선택·실행합니다.

## 사용법

```
/gadfly-method --project <프로젝트명>
/gadfly-method <연구질문 또는 주제>
/gadfly-method <연구질문> --project <프로젝트명>
```

## 하위 기능 트리

| 하위 기능 | 트리거 조건 | 배치 에이전트 | 내부 레퍼런스 |
|-----------|------------|-------------|--------------|
| 방법론 추천 | RQ에 맞는 방법론 요청 | methodology | `commands/internal/methodology.md` |
| 실험 설계 | 구체적 실험 절차 설계 | methodology + planning | `commands/internal/methodology.md` |
| IRB/윤리 | IRB, 동의서, 윤리 관련 | methodology | `commands/internal/methodology.md` |
| 설문/도구 설계 | 설문지, 측정 도구 개발 | methodology | `commands/internal/methodology.md` |
| 시스템 구현 전략 | 시스템 개발 포함 연구 | **systems-architect** + methodology | — (신규) |

## 실행 절차

### Phase 1: Director — 대화형 인터뷰

프로젝트 설정이 있으면 먼저 읽고, 없거나 불충분한 정보는 AskUserQuestion으로 대화형 수집합니다:

1. **연구 질문**: "어떤 연구 질문에 답하고 싶으신가요?"
2. **연구 목적**: "탐색적/기술적/설명적/인과적 연구 중 어디에 해당하나요?"
3. **대상/맥락**: "연구 대상은 누구이고, 어떤 맥락인가요?"
4. **데이터 접근성**: "어떤 데이터를 수집할 수 있나요? (설문, 인터뷰, 관찰, 기존 데이터 등)"
5. **제약 조건**: "시간, 예산, 참여자 접근성 등 제약이 있나요?"

### Phase 2: Director — 연구 유형 판단

인터뷰 결과를 바탕으로 연구 유형을 판단합니다:

- **양적 연구**: 실험/준실험/조사 → 방법론 추천 + 설문/도구 설계
- **질적 연구**: 사례연구/문화기술지/현상학 → 방법론 추천 + 인터뷰 프로토콜
- **혼합 연구**: 양적+질적 → 혼합 설계 전략
- **DBR(설계기반연구)**: 반복 설계 사이클 → 방법론 + 시스템 구현
- **시스템 연구**: 시스템/도구/플랫폼 개발 포함 → **systems-architect 추가 배치**

**시스템 연구 감지 키워드**: "시스템", "플랫폼", "개발", "구현", "아키텍처", "AI 시스템", "프로토타입", "앱", "웹", "챗봇", "LLM", "모델", "파이프라인"

### Phase 3: 방법론 전문가 배치

**방법론전문가(gadfly-methodology)**에게 Task를 지시합니다:

- 프로젝트 설정 + 인터뷰 결과 전달
- `knowledge/core/research-methodology.md` 참조 지시
- `knowledge/core/survey-design.md` 참조 지시 (설문 포함 시)
- `knowledge/core/statistical-methods.md` 참조 지시
- 이론↔방법론 정합성 검증 요청
- IRB 체크리스트 포함 요청
- 재현가능성 확보 수준 상세 기술 요청

### Phase 4: 시스템 구현 전략 (시스템 연구 감지 시)

**시스템 설계 전문가(gadfly-systems-architect)**를 추가 배치합니다:

- 연구 방법론과 정합되는 시스템 구현 전략 수립
- 기술 스택이 연구 목적에 부합하는지 검증
- 프로토타입 개발 로드맵 (연구 일정과 연동)
- DBR 연구 시: 설계-개발-평가 사이클 구체화
- `knowledge/domains/ai-systems/` 참조 (AI 시스템 시)

**방법론전문가(gadfly-methodology)**와 **시스템 설계 전문가(gadfly-systems-architect)**의 결과를 대조하여 정합성을 확인합니다.
